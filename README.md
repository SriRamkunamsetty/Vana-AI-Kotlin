## 9. LOCAL AI ENGINE

VANA AI hosts an advanced Local AI Orchestration pipeline:

```
                              [ PROMPT SUBMITTED ]
                                       │
                                       ▼
                     Is Ollama Localhost Operational? (TCP 11434)
                        /                             \
                       YES                             NO
                      /                                 \
          [ Ollama Gemma Streaming ]            [ Procedural AI NLP Engine ]
          - JSON Stream chunk decoding           - Deterministic parsing
          - Dynamic token generation             - On-device dictionary lookup
          - Under 50ms latency                   - Hardware-throttled delays
```

### Ollama & Gemma Pipeline
The engine targets a native port of Google's lightweight `gemma` model hosted on the Android SoC or connected via local OTG network relays.
- **REST Streaming Reader**: Standardizes an OkHttp3 connection directly to `localhost/api/generate` on the Android emulator or device looping framework.
- **Chucked Buffering**: Operates a `BufferedReader` over incoming chunked JSON byte streams. Decodes individual token entries progressively, avoiding memory-intensive batch requests.
- **Procedural Resilience**: If the socket is unreachable, the system activates a local procedural fallback engine. It employs high-density pattern detection to deliver contextual answers based on raw manual databases, ensuring advice can be generated without any server, connection, or credentials.

## 13. PERFORMANCE OPTIMIZATION

Our core runtime engine is designed to prevent UI threading lag, which can be critical during high-stress situations:

* **Jetpack Compose Layout Isolation**: Components are built with isolated sub-compositions, ensuring telemetry data updates (Compass angles, step inputs) do NOT trigger recompositions of static menus or outer containers.
* **Optimized Modifier Reuse**: Prevents allocation of fresh `Modifier` objects in fast draw blocks. Local canvas indicators use static layouts to achieve fluid 120fps rendering.
* **Background Thread Dispatcher**: Heavy operations (SQLite reads/saves, Ollama Stream parsing) are offloaded to `Dispatchers.IO`, keeping standard main threads dedicated to screen interactivity.

## 16. INSTALLATION GUIDE

Ensure you have a modern Android Development machine prior to setup:

### Prerequisites
* **Android Studio Ladybug** (2024.2.1) or higher.
* **Java Development Kit (JDK) 17** configured in your compilation terminal.
* **Android SDK v34** packages installed.
* **A physical hardware device** (highly recommended for compiling Compass, Camera, and Beacon sensors).

### Step-by-Step Compilation

1. **Clone the Repository**:
   ```bash
   git clone https://github.com/aistudio/vana-ai.git
   cd vana-ai
   ```

2. **Configure Environment variables**:
   Create a `.env` file at the root repository directory to wire optional variables (such as safe secure Gemini Fallbacks):
   ```properties
   GEMINI_API_KEY=your_secured_cloud_api_key_here
   ```

3. **Open Project in Android Studio**:
   - Select **File > Open** and locate the cloned folder.
   - Let Gradle import the project resources.

4. **Sync Project**:
   Click **Sync Project with Gradle Files** in the main toolbar.

5. **Deploy the application**:
   - Connect your Android target via USB.
   - Click the green **Run** arrow or run:
     ```bash
     gradle assembleDebug
     ```

## 17. LOCAL AI SETUP

To run VANA AI alongside a real local LLM, follow these configurations:

### Setting Up Ollama

1. **Download Ollama**:
   Download and install the correct binary for your host operating system (macOS, Linux, or Windows) from [Ollama.com](https://ollama.com).

2. **Pull Gemma model**:
   Execute the following command inside your host environment terminal to pull the micro Gemma model:
   ```bash
   ollama pull gemma
   ```

3. **Start Local Server with Cross-Origin Access**:
   You must start Ollama specifying proper environment permissions so the Android emulator sandbox can pass REST commands:
   - **For macOS/Linux**:
     ```bash
     OLLAMA_ORIGINS="*" OLLAMA_HOST="0.0.0.0" ollama serve
     ```
   - **For Windows**:
     Close Ollama from the system tray, then launch a PowerShell window and run:
     ```powershell
     $env:OLLAMA_ORIGINS="*"
     $env:OLLAMA_HOST="0.0.0.0"
     ollama serve
     ```

### Connecting Your Android Device / Emulator

- **If using the Android Emulator**:
  The emulator maps your host machine’s virtual loopback address automatically to `10.0.2.2`. VANA AI targets this host by default, enabling instant out-of-the-box local streaming.
- **If using a Physical Android Device**:
  Ensure both the developer computer and the Android handset are sharing the same Local Wi-Fi router. Inside Android Studio’s Terminal, run:
  ```bash
  adb reverse tcp:11434 tcp:11434
  ```
  This routes local traffic from port `11434` on your handset back to the Ollama server on your computer, eliminating configuration issues.
